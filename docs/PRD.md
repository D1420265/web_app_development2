# 產品需求文件 (PRD) - 英文單字系統

## 1. 專案概述
- **背景與動機**：在英語學習過程中，學生常需要快速查詢單字並記錄起來，老師也希望能提供一個簡單的工具協助學生學習。本系統旨在提供一個輕量、易用的英文單字查詢與管理平台。
- **目標用戶**：學生和老師。
- **核心價值主張**：提供快速的中英雙向查詢、翻譯功能，並協助使用者輕鬆記錄與學習英文單字。

## 2. 功能需求
以下為系統主要功能及其使用者故事：

1. **輸入與新增單字**
   - 作為 [學生/老師]，我希望 [能夠手動輸入並新增中文與英文單字]，以便 [建立專屬的單字庫]。
2. **查詢單字**
   - 作為 [學生/老師]，我希望 [能夠透過關鍵字快速查詢系統內的單字]，以便 [確認單字的拼寫與意義]。
3. **翻譯功能**
   - 作為 [學生]，我希望 [能夠輸入中文或英文並獲得對應的翻譯]，以便 [在閱讀或寫作時快速獲得解答]。
4. **個人單字本 (收藏單字)**
   - 作為 [學生]，我希望 [能夠將查詢過的單字加入收藏清單]，以便 [日後集中複習與加強記憶]。
5. **單字測驗與複習**
   - 作為 [學生]，我希望 [系統能提供簡單的單字測驗功能]，以便 [檢驗自己的學習成效]。

## 3. 非功能需求
- **技術限制**：使用 Flask (Python 後端) + Jinja2 (前端模板引擎) + SQLite (資料庫) 進行開發。
- **效能考量**：系統需能快速回應查詢請求，確保流暢的操作體驗。
- **安全考量**：防範基本的 SQL Injection 與 XSS 攻擊，確保使用者輸入的資料能被安全處理。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 輸入與新增中文、英文單字
  - 基本單字查詢與列表顯示
  - 中英雙向翻譯顯示
- **Should Have (應該有)**
  - 個人單字本收藏功能
- **Nice to Have (可以有)**
  - 單字測驗/複習功能
  - 語音發音功能 (串接外部 API)
